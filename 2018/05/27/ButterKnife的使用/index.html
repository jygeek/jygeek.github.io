ButterKnife的使用

发表于 2018-05-27 | 分类于 Android开发

简介

ButterKnife是一种用于控件初始化的注解框架，使用它可以为我们减少很多工作量，提高开发效率,下面来讲讲它的基本配置和使用

1.下载ButterKnifeZelezny插件

依次在AS的工具栏中点击File->Settings出现下图

由于我的已经下载过了，所以右上方出现的是Uninstall按钮，第一次下载时显示的是Install按钮，点击安装，然后重启一下AS就完成安装了

3.配置一次，高枕无忧

上面的方法只是针对某一个项目而言可以使用了，当再次新建一个项目时又需要重新配置，不过办法总比问题多，我们可以在AS的安装目录下先找到AndroidStudio\plugins\android\lib\templates\gradle-projects\NewAndroidModule\root\build.gradle.ftl文件，然后在dependency mavenUrl=”com.android.support:appcompat-v7:${buildApi}.+”/下方加上
<dependency mavenUrl="com.jakewharton:butterknife:8.4.0"/>
因版本不同，可能有的人到这一步还是不行，需要加上以下三条
classpath ‘com.neenbedankt.gradle.plugins:android-apt:1.8’
apply plugin: ‘android-apt’
apt ‘com.jakewharton:butterknife-compiler:8.4.0’
至于位置跟gradle文件中是类似的，这点我没有测试，因为我的在上一步就出来了,

4.小结

在第一次看别人的一个项目源码时看到bindview这个注解，然后开始研究ButterKnife,当时配置的时候踩了不少坑，这次AS升级后这个框架又不能使用了，借着这次机会找到了一种配置一次就能一直使用的办法，在写这篇博客时其中有不少是参照一篇大牛的博客的，在这里表示感谢，让我少踩不少坑，不过也有些是我自己实践的经验，ButterKnife除了给按钮初始化也可以给控件加监听等一些其他的，大家感兴趣的可以继续去深究一下
